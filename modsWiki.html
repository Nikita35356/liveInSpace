Теперь когда вы знаете что вам нужно начнём создание.

1. Создать папку мода (без разницы где)
2. Обязательно создать в них main.json и description.json, их заполнение в следущем пункте.

Главные файлы

Файл main.json и description.json являются «корневыми файлами» . Без них невозможно создать мод . Сначала разберём description.json . Пример того, что в нём находиться.
{ "name":{ "ru":"Ванила", "en":"Vanilla" }, "description":{ "ru":"Это копия ванильного мода для изучения основ мододеланья", "en":"This is a copy of a vanilla mod for learning the basics of modding." } }
В «name» находиться название мода. В «description» описание мода. Подробнее о «ru» и «en» смотреть в локализация Файл main.json Также существует и main файл. В нём уже описываются файлы мода. Пример:
[ ["items.json","items"], ["texturesItems.json","textures"], ["texturesBuilds.json","textures"], ["options.json","options"], ["names.json","names"], ["builds.json","builds"], ["loot_table.json","loot_table"] ]
В массиве перечисляются все файлы. Шаблонный элемент:
["items.json","items"]
"items.json" это абсолютный путь к файлу от папки мода. "items" это тип файла. Подробнее здесь

Типы файлов

В main.json файле есть тип файлов, но зачем они? Тип даёт понять про что будет файл, с текстурами или локализацией. Есть много типов: • names – файл локализации • textures – файл со списком текстур • items – файл со списком предметов • builds – файл со списком строений • loot_table – файл со списком лутов • options – файл со списком намтроек (в разработке) • ui – файл с интерфейсом (в разработке) • scripts – файл со скриптами (в разработке) Все типы маленькими буквами

Файл предметов

Вот уже начинаем создавать предметы. Для начала создаём файл json. Для удобства назовём items.json (не обязательно) Как только мы создали файл, добавляем в main.json
["путь к файлу от папки мода","items"],
Открываем только что созданный файл (items.json). Пример того что в нём будет:
{ "iron_ore":{ "name":"iron_ore", "texture": "iron_ore" }, "iron_ingot":{ "name":"iron_ingot", "texture":"iron_ingot" } }
Шаблонный элемент:
"id_item":{ "id_name":"iron_ore", "id_texture": "iron_ore" },
«id_item» – айди предмета для будущей работы с ним «id_name» - айди локализации «id_texture» - айди текстуры

Текстуры

Для мода надо текстуры? То создайте json файл (например texture.json) И по традиции добавте в main.json шаблон изменив путь и тип. Пример файла с текстурами
{ "space_stone":{ "type":"image", "path":"textureItem/space_stone.png" }, "dovynaid":{ "type":"animation", "format":"png", "path":[ "textureItem/dovynaid/1.png", "textureItem/dovynaid/2.png", "textureItem/dovynaid/3.png", "textureItem/dovynaid/4.png" ] } }
Статичная текстура
"id_texture":{ "type":"image", "path":"textureItem/space_stone.png" },
Анимированная текстура
"id_texture":{ "type":"animation", "format":"png", "path":[ "textureItem/dovynaid/1.png", "noName/dovynaid/2.png", "textureItem/randomItem/3.png", "gameOver/bobr/228.png" ] }
Как видно различия есть. У статичной картинки type равен "image", а у анимации "animation". У статичной текстуры path равен путю к картинке, а у анимации массивом с путями к картинкам.

Постройки

Создаём файл, добавляем в main.json по старой схеме. Пример:
{ "id_build"{ "name":"id_localization", "live":30, "loot_table":"id_loot_table", "texture":"id_texture" } }
"id_build" - айди постройки "name" - айди локализации "live" - прочность "loot_table" - айди таблицы лутов "texture" - айди текстуры
